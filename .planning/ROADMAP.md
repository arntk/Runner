# Roadmap: Runners

## Phases

- [ ] **Phase 1: User Profile & Foundation** - Set up athlete identity and baseline training zones.
- [ ] **Phase 2: Real-time Data Sync & Ingestion** - Implement Strava webhooks and contextual post-run check-ins.
- [ ] **Phase 3: Core Analytics (Zones & Clustering)** - Apply ML to categorize activities and anchor zones with manual data.
- [ ] **Phase 4: Advanced Training Metrics** - Implement Norwegian-specific load tracking and hybrid race predictions.
- [ ] **Phase 5: AI Coaching Experience** - Deliver personalized GPT-4o feedback and training summaries.
- [ ] **Phase 6: Performance Logistics (Gear & Nutrition)** - Automate weather-based gear prep and recurring nutrition protocols.

---

## Phase Details

### Phase 1: User Profile & Foundation
**Goal**: User can define their training parameters (HR/Pace zones) so analytics have a baseline.
**Depends on**: Nothing
**Requirements**: FOUND-03
**Success Criteria**:
  1. User can create/edit a profile with custom HR and Pace zones (Zones 1-5).
  2. System persists and retrieves user zones for use in analytics.
**Plans**:
  - [ ] Plan 01-01: Multi-User Backend Foundation (Wave 1)
  - [ ] Plan 01-02: Analytics Intelligence & VDOT Service (Wave 1)
  - [ ] Plan 01-03: Mobile Authentication & Onboarding (Wave 2)
  - [ ] Plan 01-04: Mobile Profile & Zone Management (Wave 2)
  - [ ] Plan 01-05: Strava Multi-User Integration & E2E Validation (Wave 3)
**UI hint**: yes

### Phase 2: Real-time Data Sync & Ingestion
**Goal**: Activities appear in the app immediately with user-provided "feel" context.
**Depends on**: Phase 1
**Requirements**: FOUND-01, FOUND-02, FOUND-04
**Success Criteria**:
  1. Strava activities are automatically synced via webhooks within 30 seconds of upload.
  2. User can rate "Feel" and RPE (1-10) via a post-run notification/interface.
  3. Raw activity data (Time, Distance, HR, Pace, Elevation) is correctly persisted in the database.
**Plans**: TBD
**UI hint**: yes

### Phase 3: Core Analytics (Zones & Clustering)
**Goal**: Automatically categorize activities by intensity using ML and manual anchors.
**Depends on**: Phase 2
**Requirements**: ANALYTIC-01, ANALYTIC-05
**Success Criteria**:
  1. User can manually enter Lactate Threshold test results to refine automatic zone calculations.
  2. K-Means clustering successfully labels activities as "Easy", "Threshold", or "Speed" based on historical data.
  3. Activity detail view displays the identified cluster and zone distribution.
**Plans**: TBD
**UI hint**: yes

### Phase 4: Advanced Training Metrics
**Goal**: Specialized insights for Norwegian method training and race readiness.
**Depends on**: Phase 3
**Requirements**: ANALYTIC-02, ANALYTIC-03, ANALYTIC-04
**Success Criteria**:
  1. System identifies "Double Threshold" days and displays aggregated "Daily Load".
  2. System detects "Intensity Creep" in threshold intervals (loss of pace/HR consistency) and flags it.
  3. Users can view race predictions generated by the hybrid Riegel/VDOT/K-Means model.
**Plans**: TBD
**UI hint**: yes

### Phase 5: AI Coaching Experience
**Goal**: Users receive personalized, plain-English training feedback and strategic guidance.
**Depends on**: Phase 4
**Requirements**: COACH-01, COACH-02, COACH-03, COACH-04
**Success Criteria**:
  1. User receives a GPT-4o powered "Benefit" explanation immediately after run sync.
  2. AI provides feedback (reprimand/encouragement) if RPE conflicts with objective intensity data.
  3. User can view a weekly summary that suggests training volume/intensity adjustments.
  4. User can activate "Race-Peaking Mode" to focus analytics on a specific upcoming goal.
**Plans**: TBD
**UI hint**: yes

### Phase 6: Performance Logistics (Gear & Nutrition)
**Goal**: Users are prepared for every run with the right gear and nutrition protocols.
**Depends on**: Phase 2
**Requirements**: LOGISTIC-01, LOGISTIC-02, LOGISTIC-03
**Success Criteria**:
  1. User receives gear suggestions based on real-time weather and the day's planned intensity.
  2. User can initiate and follow a 48-hour nutrition/carb-loading countdown.
  3. User receives daily push notifications for supplement checklists (Iron, Magnesium, etc.).
