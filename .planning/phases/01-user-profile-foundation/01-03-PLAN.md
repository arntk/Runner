# Plan 01-03: Mobile Auth & Onboarding UI (Wave 2)

**Goal**: Implement the mobile user interface for account creation, login, and initial training zone setup.

## Authentication Interface

### Task 01-03-01: Login & Registration Screens
- <read_first> `mobile/app/auth.tsx`
- <action> Implement `Login` and `Register` functional components in `mobile/app/auth.tsx`. Use `View`, `Text`, and `TextInput` primitives. Set background color to `#0a0f1e`, input background to `#111827`, and text to white. Ensure inputs have `autoCapitalize="none"` for email.
- <acceptance_criteria> `grep -r "TextInput" mobile/app/auth.tsx` returns matches; the components render in the emulator with the correct colors.

### Task 01-03-02: API Integration (JWT)
- <read_first> `mobile/services/api.ts`
- <action> Install `@react-native-async-storage/async-storage`. Implement `login(email, password)` and `register(email, password)` functions using `fetch`. The functions must persist the received JWT in AsyncStorage using the key `userToken`.
- <acceptance_criteria> `grep -r "AsyncStorage.setItem('userToken'" mobile/services/api.ts` returns matches.

## Onboarding Flow

### Task 01-03-03: Step 1 - Biological Data
- <read_first> `.planning/phases/01-user-profile-foundation/01-UI-SPEC.md`
- <action> Create `mobile/app/onboarding/step1-bio.tsx`. Implement form inputs for `weight_kg`, `max_hr`, and `resting_hr`. Use numeric keyboards for all stats. Apply 16px (md token) padding between input rows.
- <acceptance_criteria> `grep -r "keyboardType='numeric'" mobile/app/onboarding/step1-bio.tsx` returns matches.

### Task 01-03-04: Step 2 - Zone Calibration (VDOT)
- <read_first> `.planning/phases/01-user-profile-foundation/01-CONTEXT.md` (Decision D-08)
- <action> Implement `mobile/app/onboarding/step2-zones.tsx`. Include inputs for a recent race (Distance, Time) to fetch VDOT estimates from the Python backend. Add a "Unit Preference" toggle for Heart Rate (BPM vs % Max HR) and Pace (min/km vs % Threshold Pace). Display the 5 estimated zones using `ZonePill` components.
- <acceptance_criteria> `grep -r "% Max HR" mobile/app/onboarding/step2-zones.tsx` and `grep -r "ZonePill" mobile/app/onboarding/step2-zones.tsx` return matches.

### Task 01-03-05: Finalize Onboarding
- <read_first> `mobile/app/_layout.tsx`
- <action> Update the root layout navigation. Implement an `useEffect` hook to check for `userToken`. If no token, redirect to `/auth`. If token exists, check if profile is complete (via `GET /api/profile`). Redirect to `/onboarding` if incomplete, or `/coach` if complete.
- <acceptance_criteria> Navigation logic successfully routes to `/onboarding` upon first login with a new account.
