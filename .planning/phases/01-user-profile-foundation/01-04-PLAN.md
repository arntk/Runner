# Plan 01-04: Mobile Profile & Zone Management (Wave 2)

**Goal**: Implement a dedicated Profile tab for users to manage their athlete identity and training parameters.

## Profile Interface

### Task 01-04-01: Create Profile Tab
- <read_first> `mobile/app/(tabs)/_layout.tsx`
- <action> Add a new tab `(tabs)/profile.tsx` to the `Tabs` layout. Use `User` icon from `lucide-react-native`. Apply the primary accent color `#6366f1` to the active tab state and set the tab bar background to `#111827`.
- <acceptance_criteria> A "Profile" tab icon is visible in the bottom navigation bar and navigates to the (currently blank) profile screen.

### Task 01-04-02: Profile Header & Info
- <read_first> `mobile/app/(tabs)/profile.tsx`
- <action> Implement the Profile screen layout using `ScrollView`. Create a header section with `Text` for the user's name (20px, bold) and email (16px, regular). Add a "Bio Stats" card showing `weight_kg`, `max_hr`, and `resting_hr` with edit icons.
- <acceptance_criteria> User stats are fetched from `GET /api/profile` and correctly displayed in the UI.

## Zone Management

### Task 01-04-03: Training Zone List
- <read_first> `.planning/phases/01-user-profile-foundation/01-CONTEXT.md` (Decision D-08)
- <action> Implement a "Training Zones" section in the Profile tab. Use a `FlatList` or `SectionList` to display the 5 Heart Rate and 5 Pace zones. Use `ZonePill` components to display each zone's range. Add a toggle button to switch the entire list view between absolute (BPM/min-km) and relative (% Max/% Threshold) units.
- <acceptance_criteria> `grep -r "ZonePill" mobile/app/(tabs)/profile.tsx` returns matches; the unit toggle successfully updates the displayed values and units.

### Task 01-04-04: Zone Editing (Manual Overrides)
- <read_first> `mobile/app/(tabs)/profile.tsx`
- <action> Allow users to tap on any zone to open a modal for manual editing. The modal must provide an input field and a unit selector (e.g., "BPM" vs "% of Max HR"). Implement a "Save" button that performs a `PUT /api/profile` call with the updated zone JSON. Add a "Reset to Calibrated" button that clears manual overrides and re-fetches estimates from the VDOT service.
- <acceptance_criteria> Manual edits to zones are correctly persisted to the backend; `grep -r "modal" mobile/app/(tabs)/profile.tsx` returns matches.
